# takeAway — Project Brief

## Концепция
**takeAway** — глобальная **pre-order first** платформа для кофе и еды навынос. Клиент делает предзаказ заранее, следит за статусом приготовления, приходит в нужное время и забирает готовый заказ без очереди. Идея вдохновлена Drinkit (Dodo Brands), но расширена на завтраки, ланчи и готовую еду.

Ключевой ценностный тезис: **«Pre-order. Skip the queue. Pick it up.»** — клиент экономит главное: время.

## Проблема, которую решаем
Люди теряют 10–20 минут в очереди каждое утро за кофе и едой. В час пик это критично для опоздания на работу, встречу или занятие. Существующие приложения (Starbucks, кофейни) часто лишь дублируют обычный заказ в приложение, не меняя механику: пользователь всё равно стоит и ждёт.

**takeAway меняет логику**: заказ готовится к моменту прихода. Пользователь **никогда не ждёт у прилавка**.

## Бизнес-модель
- **B2C**: продажа кофе и еды через предзаказ в приложении/вебе/Telegram
- **B2B (v2)**: SaaS / франшиза для независимых кофеен и tech-кафе, которые используют нашу платформу как white-label pre-order систему
- **Подписки**: Coffeepass-style — N напитков в день/неделю за фикс (high-margin retention)
- **Комиссия маркетплейса (v3)**: если захотим открыть платформу другим заведениям

## Целевая аудитория
- **Primary**: занятые городские профессионалы 22–40, ценят скорость и цифровой UX
- **Secondary**: студенты, удалёнщики, цифровые кочевники
- **География**: глобальный запуск, старт с англоязычных рынков (Dubai, London, Lisbon, Bangkok, Bali)

## Ключевые принципы продукта
1. **Pre-order first**: каждый заказ — это предзаказ. Клиент указывает когда придёт, мы готовим к этому моменту. Никаких очередей, никаких касс на месте
2. **Time-to-pickup transparency**: клиент всегда видит точный ETA и статус в реальном времени (Received → Preparing → Ready → Picked up). Готовый заказ ждёт клиента, а не наоборот
3. **Smart pickup**: QR-код / 4-значный код для получения, геофенсинг «я рядом», ячейки с замком (pickup lockers) или smart counter в v2
4. **Deep customization**: любой напиток/блюдо конфигурируется до последней детали
5. **Smart recommendations**: AI-подсказки "обычный заказ", "закажи как в прошлый вторник"
6. **Omnichannel**: iOS + Android + Web/PWA + Telegram Mini App — одна корзина, один профиль, один заказ виден везде
7. **Loyalty-first**: геймификация, подписки, рефералка встроены с первого дня

## Основной пользовательский сценарий (happy path)
1. **За 5–15 минут до пункта назначения** клиент открывает приложение
2. Приложение показывает ближайшие точки и **когда заказ будет готов** если оформить сейчас
3. Клиент выбирает товары, кастомизирует, указывает **время получения** (ASAP или на конкретное время)
4. Оплачивает → получает **order code + QR + push с ETA**
5. В пути видит live-таймер и статус: Received → Preparing → Ready
6. **При приближении к точке** приложение само уведомляет бариста: «клиент в пути»
7. Клиент приходит, показывает код, **забирает готовый заказ из именного отсека** — ноль взаимодействий с кассой
8. Получает чек, бонусные баллы, push-подсказку "повторить завтра"

## Чем отличаемся от Drinkit
| Параметр | Drinkit | takeAway |
|----------|---------|----------|
| Главная механика | Order → pickup в зале | **Pre-order first с ETA и live-статусом** |
| Ассортимент | Только напитки | Напитки + завтраки + ланчи + десерты |
| Каналы | Нативное приложение | iOS + Android + Web/PWA + Telegram Mini App |
| Рынок | MENA (Дубай) | Глобальный с первого дня |
| Оплата | Локальные + Apple Pay | Stripe + Apple/Google Pay + Telegram Payments |
| Подписка | Coffeepass | Coffeepass + FoodPass + комбо-подписки |
| Открытость | Закрытая экосистема | White-label pre-order для партнёров (v2) |
| Pickup UX | Smart counter в зале | Именной отсек / QR / геофенсинг «я в пути» |
| Доставка | Нет | Вторичная опция в v1.5, не core |

## Решения, принятые до начала работ
- **Концепция**: **pre-order first** платформа для кофе + еды навынос (не только напитки, не dine-in, не delivery-first)
- **Главная фича**: предзаказ с ETA и live-отслеживанием статуса, smart pickup без очереди
- **Доставка**: существует как опциональная фича, но НЕ центральная в MVP. Основной flow — самовывоз
- **Платформы MVP**: Web/PWA + Telegram Mini App + нативные iOS/Android
- **Рынок**: глобальный, Stripe как основной процессинг
- **Стек**: TypeScript монорепо (Angular + NestJS), Flutter для мобильных приложений в v2
- **Язык общения в команде**: русский, язык кода/коммитов/документации — английский
